## 快速开始

### 30 秒上手

1. **在 Home 首页记录想法**：使用首页的 Quick Capture 快速记录任何灵感
2. **去 Inbox 澄清想法**：让 AI 帮你整理模糊的想法，生成标题、摘要和行动建议
3. **创建 Project 并生成 Tasks**：把澄清后的想法变成可执行的任务清单
4. **在 Tasks 中推进工作**：管理任务状态，查看今日和本周待办
5. **用素材沉淀内容**：在 Vault 收集和管理相关素材
6. **生成最终草稿**：在 Studio 基于想法和素材生成创作内容

## 核心工作流

Echive 围绕个人创作者的完整创作流程设计：

```
💡 想法输入 → 🤖 AI 澄清 → 📋 任务拆解 → 📚 素材沉淀 → ✍️ 草稿生成
```

每个阶段对应一个独立的模块，数据在模块间无缝流动。

## 模块详解

### Home 首页

首页是你的工作台仪表盘，展示当前工作流的全景：

#### 功能概览

| 区域 | 说明 |
|------|------|
| **数据统计卡片** | 一眼看到：今日任务数、总想法数、待澄清数、草稿总数 |
| **Quick Capture** | 随时记录闪现的灵感，无需离开首页 |
| **今日任务** | 今天需要完成的任务清单，带状态和优先级 |
| **最近想法** | 最新记录的 5 个想法，点击可进入详情 |
| **最近草稿** | 最新编辑的草稿，继续你的创作 |
| **待澄清横条** | 高亮显示需要处理的模糊想法 |

#### 使用建议
- 养成随时记录的习惯：想到什么就用 Quick Capture 记下来
- 每天早晨查看「今日任务」，规划一天的工作
- 看到「待澄清横条」中有内容时，优先处理

---

### Inbox 想法箱

Inbox 是所有想法的入口和中转站。这里的目标是把模糊的想法变清晰。

#### 功能概览

| 功能 | 说明 |
|------|------|
| **Quick Capture** | 快速记录想法，自动生成默认标题 |
| **想法列表** | 查看所有想法，可按状态筛选 |
| **AI 澄清** | 让 AI 帮你整理想法：生成标题、摘要、澄清问题和下一步行动 |
| **状态流转** | CAPTURED（待澄清）→ CLARIFIED（已澄清）→ PLANNED（已规划） |

#### 澄清流程

1. 点击任意想法进入详情页
2. 点击「开始澄清」按钮
3. AI 会自动分析并生成：
   - **建议标题**：简洁明了的概括
   - **摘要**：用一两句话描述这个想法
   - **澄清问题**：帮你把想法想透彻的关键问题
   - **下一步行动**：这个想法接下来可以做什么
4. 你可以编辑 AI 生成的内容，或重新生成
5. 确认后，想法状态变为「已澄清」

#### 状态说明

| 状态 | 含义 | 下一步 |
|------|------|--------|
| CAPTURED | 刚记录的原始想法，等待澄清 | 点击开始澄清 |
| CLARIFIED | 已澄清，等待规划 | 关联到 Project 并生成 Task |
| PLANNED | 已生成任务，正在执行中 | 推进任务完成 |
| ARCHIVED | 已归档 | 可随时恢复 |

### Projects 项目管理

Project 是 Idea 和 Task 的容器，帮你把相关的工作组织在一起。

#### 功能概览

| 功能 | 说明 |
|------|------|
| **创建项目** | 为你的创作目标创建一个项目 |
| **项目列表** | 查看所有项目，可按状态筛选 |
| **项目详情** | 查看项目下关联的所有想法和任务 |
| **关联 Idea** | 把已澄清的想法添加到项目中 |

#### 使用建议

- 一个创作目标对应一个 Project
- 可以把多个相关的 Idea 关联到同一个 Project
- 在项目详情页可以看到整体进度

---

### Tasks 任务管理

把大想法拆解成可执行的小任务，逐步推进完成。

#### 功能概览

| 功能 | 说明 |
|------|------|
| **任务列表** | 查看所有任务，支持多维度筛选 |
| **状态切换** | TODO → DOING → BLOCKED → DONE，一键切换 |
| **优先级设置** | HIGH / MEDIUM / LOW，明确任务重要性 |
| **截止日期** | 设置任务完成时间 |
| **今日视图** | 只看今天需要完成的任务 |
| **本周视图** | 查看本周的任务安排 |
| **按项目筛选** | 查看某个项目下的所有任务 |

#### 任务生成

从已澄清的 Idea 可以一键生成任务：
1. 进入已澄清的 Idea 详情页
2. 点击「生成任务」
3. AI 会自动生成 5 个标准任务：
   - 明确目标与验收标准（高优先级）
   - 收集必要素材（中优先级）
   - 拆解最小可行版本（高优先级）
   - 完成第一轮验证（高优先级）
   - 复盘并沉淀记录（低优先级）
4. 你可以编辑、添加或删除任务
5. 任务会自动关联到对应的 Idea 和 Project

#### 状态说明

| 状态 | 含义 |
|------|------|
| TODO | 待开始 |
| DOING | 进行中 |
| BLOCKED | 遇到阻塞，需要先解决依赖 |
| DONE | 已完成 |

---

### Vault 素材库

沉淀创作过程中收集和产生的素材，方便后续复用和参考。

#### 功能概览

| 功能 | 说明 |
|------|------|
| **添加素材** | 记录任何有用的内容：文字片段、参考链接、笔记等 |
| **素材类型** | 支持多种类型：文本、链接、图片描述等 |
| **关联 Idea** | 素材可以关联到对应的想法 |
| **标签系统** | 用标签分类管理素材 |
| **搜索** | 快速找到需要的素材 |

#### 使用建议
- 看到好的参考内容就记录下来
- 素材尽量关联到对应的 Idea，方便生成草稿时使用
- 用标签做分类，比如「技术」「设计」「文案」等

---

### Studio 草稿工作室

基于已澄清的想法和收集的素材，生成完整的创作内容。

#### 功能概览

| 功能 | 说明 |
|------|------|
| **选择来源** | 选择一个 Idea 作为创作基础 |
| **选择素材** | 勾选相关素材作为参考 |
| **选择输出格式** | 支持：博客文章、微博推文、Vlog 脚本、播客脚本等 |
| **AI 生成内容** | 一键生成完整草稿，包含大纲和正文 |
| **在线编辑** | 在编辑器中修改和完善草稿 |
| **版本历史** | 查看和对比不同版本的草稿 |

#### 生成流程

1. 进入 Studio，点击「新建草稿」
2. 选择一个已规划的 Idea 作为主题
3. 勾选相关的素材（可选但推荐）
4. 选择输出格式
5. 点击生成，AI 会基于 Idea 和素材创建完整草稿
6. 在编辑器中修改和完善

## 完整创作流程演示

让我们用「写一篇关于 AI 写作工具的博客」为例，走一遍完整流程：

### 第 1 步：记录想法
在 Home 或 Inbox 的 Quick Capture 中输入：
```
想写一篇关于 AI 写作工具的文章，对比现在主流的几个工具，分析各自的优缺点，给创作者一些实用建议。
```

### 第 2 步：AI 澄清
1. 点击刚创建的想法进入详情页
2. 点击「开始澄清」
3. AI 会生成：
   - **标题**：《AI 写作工具对比分析与创作者实用指南》
   - **摘要**：对比主流 AI 写作工具的优缺点，为创作者提供选型和使用建议
   - **澄清问题**：重点对比哪些工具？目标读者是新手还是进阶用户？文章侧重功能对比还是实际使用技巧？
   - **下一步行动**：收集各工具的使用体验，确定文章结构和重点

### 第 3 步：创建项目并生成任务
1. 在 Idea 详情页点击「关联到项目」
2. 创建新项目：「AI 写作工具对比博客」
3. 点击「生成任务」，AI 自动创建 5 个任务
4. Idea 状态自动变为 PLANNED

### 第 4 步：收集素材
1. 去 Vault 添加素材
2. 记录你对 ChatGPT、Claude、文心一言等工具的使用体验
3. 把每个素材都关联到这个 Idea

### 第 5 步：推进任务
1. 在 Tasks 页面查看今日待办
2. 按照任务清单逐步推进：
   - 明确文章目标和验收标准
   - 收集各工具的使用体验素材
   - 拆解文章结构为最小可行版本
   - 完成初稿并验证
   - 复盘沉淀

### 第 6 步：生成草稿
1. 进入 Studio，点击「新建草稿」
2. 选择「AI 写作工具对比博客」这个 Idea
3. 勾选所有相关素材
4. 选择「博客文章」格式
5. 点击生成，AI 会基于所有上下文创建完整草稿
6. 在编辑器中修改和完善

### 完成！🎉
你的第一篇用 Echive 辅助创作的内容就完成了！

## 常见问题

### Q: 我的数据存在哪里？安全吗？
A: MVP 版本的数据统一存储在阿里云的开发数据库中。正式版会提供本地优先的存储方案，你的数据永远在你的控制之下。

### Q: AI 生成的内容可以修改吗？
A: 当然可以！所有 AI 生成的内容（标题、摘要、任务、草稿）都可以编辑。AI 只是帮你打初稿，最终的决定权在你。

### Q: 可以多个 Idea 关联到同一个 Project 吗？
A: 可以的！一个 Project 可以包含多个相关的 Idea，适合处理系列内容创作。

### Q: 支持团队协作吗？
A: MVP 版本专注于单人使用场景。团队协作功能会在后续版本中考虑。

### Q: 可以导出数据吗？
A: 可以的，所有数据都支持导出为 JSON 或 Markdown 格式。

## 技巧与提示

### 💡 提升效率的小技巧

1. **快速记录不要犹豫**
   - 想到什么就马上记下来，不要担心写得不好
   - 反正后面有 AI 帮你整理和澄清

2. **定期清理 Inbox**
   - 每周固定时间集中处理待澄清的想法
   - 不要让 CAPTURED 状态的想法积压太多

3. **善用今日视图**
   - 每天早晨看一眼 Today 视图，明确今天要做什么
   - 完成任务及时切换到 DONE 状态，获得成就感

4. **素材随时记录**
   - 看到好的内容就立刻存入 Vault
   - 写草稿的时候你会感谢当时的自己

5. **让 AI 多试几次**
   - 如果第一次生成的不满意，再点一次生成
   - 多试几次，总会有惊喜

### 🔄 推荐的日常工作流

```
早晨：
  1. 打开 Home，查看今日任务
  2. 有新想法就 Quick Capture

上午：
  1. 处理待澄清的想法
  2. 推进高优先级任务

下午：
  1. 收集和整理素材
  2. 生成或编辑草稿

晚上：
  1. 回顾任务完成情况
  2. 记录明天的想法
```
