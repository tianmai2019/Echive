# 3. Echive 的 API 设计初稿

## 3.1 Ideas

### POST `/api/ideas`
创建 Idea

**request**
```json
{
  "title": "做一个个人数字管家",
  "rawInput": "我想做一个自己的数字管家，帮我管理创作工作流",
  "projectId": null,
  "type": "IDEA"
}
```

**response**
```json
{
  "id": "idea_xxx",
  "title": "做一个个人数字管家",
  "status": "CAPTURED"
}
```

---

### GET `/api/ideas`
获取 Idea 列表

query:
- `status`
- `type`
- `projectId`
- `q`

---

### GET `/api/ideas/:id`
获取 Idea 详情

---

### PATCH `/api/ideas/:id`
更新 Idea
可更新：
- title
- summary
- status
- projectId
- nextAction

---

### POST `/api/ideas/:id/clarify`
触发 AI 澄清

**response**
```json
{
  "summary": "这是一个面向个人创作工作流的数字管家项目",
  "questions": [
    "这个产品主要服务谁？",
    "第一阶段最重要的能力是什么？",
    "你希望先覆盖哪些内容输出形式？"
  ],
  "nextAction": "先明确 MVP 闭环并建立数据模型"
}
```

---

### POST `/api/ideas/:id/tasks/generate`
基于 Idea 生成任务建议

**response**
```json
{
  "tasks": [
    {
      "title": "明确产品定位",
      "priority": "HIGH"
    },
    {
      "title": "设计 MVP 模块",
      "priority": "HIGH"
    }
  ]
}
```

## 3.5 Drafts

### POST `/api/drafts/generate`
生成草稿

**request**
```json
{
  "ideaId": "idea_xxx",
  "materialIds": ["mat_1", "mat_2"],
  "format": "BLOG"
}
```

**response**
```json
{
  "id": "draft_xxx",
  "title": "为什么我需要一个自己的数字管家",
  "outline": "1. 问题背景\n2. 核心需求\n3. Echive 的方案",
  "content": "..."
}
```

### GET `/api/drafts`
获取草稿列表

### GET `/api/drafts/:id`
获取草稿详情

### PATCH `/api/drafts/:id`
更新草稿
